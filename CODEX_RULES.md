## Кратко о проекте

Проект: `inventory_service`

Назначение:

- сервис для обработки catalog/stock;
- хранение и редактирование настроек предприятий;
- интеграции с Google Drive, FTP, feed/API источниками;
- обработка заказов, webhook-ов и уведомлений;
- dropship/pricing логика;
- master catalog и balancer-процессы.

## Технический стек

- Python
- FastAPI
- SQLAlchemy async
- PostgreSQL
- Alembic
- Pydantic
- Uvicorn
- systemd
- React (`admin-panel`)
- `.env` для конфигурации

ОС разработки и эксплуатации:

- локально: macOS
- сервер: Ubuntu

## Ограничения и важный контекст

- Проект не является "чистым микросервисом"; это монолит с несколькими daemon-like процессами.
- Большая часть поведения управляется `enterprise_settings` и env-переменными.
- Изменения в `app/models.py`, `app/routes.py`, `app/services/*scheduler*`, `app/business/dropship_pipeline.py`, `app/business/master_catalog_orchestrator.py` считаются high-impact.
- Для проекта важны не только HTTP-роуты, но и cron/systemd-like сценарии.
- Нужно учитывать совместимость с существующими таблицами и миграциями.
- Нельзя советовать коммитить секреты, `.env` и временные файлы.
